# Contributing to ARPA Micro Series: F1 Mask

Thank you for your interest in improving ARPA's privacy-focused middleware. We welcome contributions that align with our mission of zero-latency, secure PII scrubbing.

## How to Contribute

1.  **Fork the Project**: Create a fork of the `arpahls/micro-f1-mask` repository.
2.  **Clone Locally**: `git clone https://github.com/your-username/micro-f1-mask.git`.
3.  **Create a Branch**: `git checkout -b feature/your-feature-name`.
4.  **Make Changes**: Improve the code, tests, or documentation.
5.  **Test**: Ensure all tests in the `tests/` directory pass.
6.  **Push**: `git push origin feature/your-feature-name`.
7.  **Pull Request**: Submit a detailed Pull Request to the `main` branch.

## Rules of Engagement

-   **Zero Emojis**: We maintain a professional, minimalist aesthetic. Do not use emojis in PR descriptions or documentation.
-   **No Real PII**: Never include real personally identifiable information in issues, PRs, or testing data.
-   **Security First**: If you find a security flaw, please follow our [Security Policy](SECURITY.md).
-   **Linting**: Ensure your code follows PEP 8 standards.

## Code of Conduct

Refer to the [Agent Code of Conduct](https://github.com/arpahls/skillware/blob/main/CODE_OF_CONDUCT.md) for our standards of interaction.
